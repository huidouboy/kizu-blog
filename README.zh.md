> 目标是兼顾灵活性与长期可维护性，同时保持极低复杂度。

🌐 在线示例：https://huidouboy.github.io/kizu-blog

---

## ✨ 为什么做这个？

现有博客方案通常分两类：

- 静态生成器（Hexo / Hugo）→ 性能好，但扩展受限
- CMS（WordPress）→ 灵活，但复杂且依赖重

**Kizu Blog 的目标是融合两者优势。**

### 核心理念：

- **内容优先**：Markdown 永远是数据源
- **静态优先**：输出纯 HTML，无需服务端
- **插件优先**：扩展能力不侵入核心
- **主题优先**：界面完全可替换
- **极简运行时**：尽量减少依赖

---

## 🚀 特性

- Markdown 内容管理
- 静态 HTML 输出
- 可扩展主题系统
- Hook 插件系统
- 可选后台（文件驱动，无数据库）
- RSS / sitemap / 标签页 / 搜索索引
- 阅读时间 / 上一篇下一篇
- 响应式默认主题
- TypeScript Monorepo 架构
- 中英文双语文档
