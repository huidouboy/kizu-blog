Theme Contract

Kizu 主题制作规范

这份规范给后台管理员和主题作者查看，不放在公开博客主导航里。公开前台应只展示博客内容、归档、关于页和账号入口。

主题必须包含

  • theme.json：声明主题名称、版本、能力、默认配置。
  • templates/：主页、文章、页面、归档和 404 模板。
  • partials/：头部、底部、弹窗等局部模板。
  • assets/：CSS、JS、图片和主题私有资源。

设计要求

  • 主站必须整洁，首屏表达博客气质，但不能变成项目说明页。
  • 站点名称、署名、描述、主题色必须来自后台设置。
  • 登录注册和评论入口由主题展示，实际权限由后端 API 判断。
  • 主题不直接读写数据库，不绕过后台接口。

完整文档也保存在项目内 docs/theme-authoring.md 和主题目录 themes/neo-journal/THEME_AUTHORING.md。
