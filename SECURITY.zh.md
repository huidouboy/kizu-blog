# 安全策略

[English](./SECURITY.md) | [简体中文](./SECURITY.zh.md)

## 支持版本

项目仍处在早期阶段。除非之后建立单独的发布分支，安全修复通常会面向最新的 `main` 分支。

## 报告漏洞

如果发现安全问题，请尽量私下报告。仓库如果暂时没有开启 private advisory，请先联系维护者，不要直接在公开 issue 中贴出可利用细节。

报告时建议包含：

- 受影响的版本或 commit
- 复现步骤
- 影响范围
- 如果你有修复建议，也可以一并说明

在修复发布前，请避免公开完整的利用载荷。

## 安全模型

生成后的站点是静态 HTML。运行公开站点时不需要 Node.js、不需要数据库，也不需要后台服务。

后台面板是可选工具，面向可信任的站点维护者。它只能写入：

- `content/posts/*.md`
- `content/pages/*.md`
- `config/site.json`
- `config/theme.json`
- `config/plugins.json`

后台认证目前是单个本地账号。密码使用 PBKDF2 哈希后存放在 `data/admin/account.json`。删除的 Markdown 文件会移动到 `data/admin/trash`。

## 不在当前范围内

项目目前不提供：

- 多用户权限系统
- SaaS 多租户隔离
- 插件沙箱
- 公共托管后台服务

插件会在构建期间作为本地项目代码执行。生产环境只应启用你信任的插件。

## 部署建议

- 公开站点只发布 `dist/`。
- 除非明确需要远程编辑，否则不要暴露后台服务。
- 保持 `data/admin/` 私有，并确保它不进入 git。
- 生产环境请正确设置 `baseUrl`。
- 启用生产插件前，请先审查插件代码。
